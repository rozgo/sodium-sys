Function sodium_sys::utils::munlock [−] [src]

pub fn munlock(mem: &[u8]) -> i32

The munlock() function should be called after locked memory is not being used any more. It will zero the bytes in the array before actually flagging the pages as swappable again. Calling memzero() prior to munlock() is thus not required.

On systems where it is supported, sodium_mlock() also wraps madvise() and advises the kernel not to include the locked memory in coredumps. ss_unlock() also undoes this additional protection.

munlock safely wraps sodium_munlock.

Examples

use sodium_sys::utils::{mlock, munlock};

let v = [0, 1, 2, 3, 4, 5, 6, 7];
assert!(mlock(&v) == 0);
assert!(munlock(&v) == 0);
assert!(v == [0; 8]);
